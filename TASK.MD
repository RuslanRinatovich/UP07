### Модуль 1. Установка и настройка SQL-сервера с автоматическим созданием пользователей и баз данных

**Задача:**  
Выберите СУБД и среду для управления инфраструктурой. Установите ядро выбранной СУБД и среду для управления инфраструктурой SQL (на виртуальную машину или представленный компьютер).

**⚙️ Требования к установке:**
- Имя сервера: `Server_номер_рабочего_места` (пример: `Server_05`)
- Режим аутентификации: **смешанный**
- Пользователь `sa`: включить или создать
- Пароль `sa`: `De_номер_рабочего_места` (пример: `De_05`)

**📝 Необходимо разработать скрипт, который:**
1. Создаёт пользователей `user1`, `user2`, …, `user10` (пароль — случайные 5 символов: буквы + цифры)
2. Создаёт базы данных `BD1`, `BD2`, …, `BD10`
3. Настраивает права доступа: `user1` → только `BD1`, `user2` → только `BD2` и т.д.
4. Создаёт базу данных `BD` и таблицу `Users` для хранения логинов и паролей
5. Заполняет таблицу `Users` данными о созданных пользователях и паролях

### Модуль 4. Проектирование и реализация базы данных на основе ER-диаграммы

**Задача:**  
На основании документов заказчика спроектировать ER-диаграмму для информационной системы.

**Описание предметной области:**  
Предприятие производит продукцию согласно установленным спецификациям и реализует готовую продукцию заказчикам. Каждая продукция имеет свою цену, зависящую от стоимости материалов.

**📌 Требования:**
- Обязательна **3 нормальная форма** с обеспечением ссылочной целостности
- Осмысленная схема именования (единый стиль, единое число)
- Создать первичные и внешние ключи
- ER-диаграмма в формате **.pdf** (таблицы, связи, атрибуты, ключи; типы данных — на данном этапе можно пренебречь)
- Заполнить таблицы тестовыми данными (не менее **3 записей** в каждую таблицу)

> ⚠️ *Внимание! Данные из ресурсов импортировать не нужно.*
